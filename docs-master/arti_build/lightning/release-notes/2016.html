2016 Release notes - Lightning¶

Looking for the latest and greatest new features and changes to Lightning? Read on and check back regularly to see what else we’ve done.

Note

  • Lightning can include released, beta, alpha, or patched modules depending on which version is determined to be the most current, stable, secure, and appropriate version for general use.
  • If you update a production website, you should always first perform a backup, and, if possible, perform a trial update in a test environment before updating the production website.

Lightning 8.x-1.14¶

14 December 2016

Lightning 8.x-1.14 (download) contains the following updates:

Module updates¶

  • Updated Acquia Connector to 8.x-1.7
  • Updated Entity API to 8.x-1.0-alpha4
  • Updated Entity Browser to 8.x-1.0-beta2
  • Updated Features to 8.x-3.2
  • Updated Media entity to 8.x-1.6

Changes¶

  • #2828638: Added Panelizer support for view mode descriptions.
  • #2827227: Tarball releases are now built with contributed versions from Drupal.org and not Git.
  • Commit #226: Updated Panelizer to fix a problem where it did not properly define its dependency on Field UI.
  • Commits #194, #223: Improved the author-facing Panelizer experience by implementing “Internal” View Modes for which Panelizer is always disabled.
  • Included a script that will convert your project’s root composer.json file to use the official Drupal.org packagist and update your project to Lightning 2.0, which also uses the official Packagist. Learn more.
  • #2834874: Lightning no longer provides default content for the Shortcut menu.
  • #2729377: Made it possible for modules and themes to pass CSS to a CKEditor instance.
Fixed issue
name:fixed-issue
  • #2831550: Having certain contributed modules installed on your Lightning website could cause an infinite loop.

Lightning 8.x-1.13¶

16 November 2016

Lightning 8.x-1.13 contains the following updates:

Module updates¶

  • Updated Entity Browser to 8.x-1.0-beta1
  • Updated Features to 8.x-3.0
  • Updated Media entity Twitter to 8.x-1.3
  • Updated Metatag to 8.x-1.0-beta11
  • Updated Multiversion to 8.x-1.0-alpha12
  • Updated Views Infinite Scroll to 8.x-1.3
  • Updated Workspace to 8.x-1.0-alpha4

Changes¶

  • Tests now ensure that composer.lock is kept up to date. (GitHub #132)
  • Twitter media entities can now be configured to generate thumbnails for text-based tweets. (GitHub #203)
  • CI now uses database snapshots for update tests. (GitHub #201)
  • Display modes and user roles can now have associated help text and descriptions. (GitHub #195)
  • The Panelizer interface was updated to improve its usability. (Issue #2826071)
  • Lightning extender can now be used to exclude submodules of Lightning extensions. (GitHub #220)

Fixed issues¶

  • The page title block was missing from the Seven theme. (GitHub #190)
  • After creating several node revisions, only the first revision was displayed in the list. (Issues #2824633 and #2825477)

Lightning 8.x-1.03¶

3 August 2016

Lightning contains the following updates:

Update note

If your website has custom Entity Browser code, you may need to update due to an Entity Browser module update that introduces several backwards-incompatible API changes. Be sure to back up your database before updating to this version of Lightning.

Feature¶

  • Easier image uploads to your site Lightning now includes an image browser that you can use to upload images to and select images from your media library, and is used by default for all new image fields. Although it is not added to existing image fields, you can manually modify existing fields to use the image browser.

Drupal core and module updates¶

  • Updated Drupal Core to 8.1.8
  • Updated Acquia Connector to 8.x-1.3
  • Updated Entity Browser to 8.x-1.0-alpha7
  • Updated Entity Embed to 8.x-1.0-alpha3
  • Updated Media entity to 8.x-1.3
  • Updated Media entity document to 8.x-1.1
  • Updated Media entity image to 8.x-1.2
  • Updated Media entity Instagram to 8.x-1.2
  • Updated Media entity Twitter to 8.x-1.2
  • Updated Video Embed Field to 8.x-1.2
  • Updated Views Infinite Scroll to 8.x-1.2

Changes¶

  • #2759825: A sanity check was added to prevent fatal errors when preparing extra fields for media asset previews.
  • #2766337: The Lightning Extender will now search for lightning.extend.yml in sites/all as well as your website’s individual directory, allowing every website in a multisite Lightning installation to use the same extender configuration.

Fixed issues¶

  • #2768849: The media browser would not appear when editing a node with an embedded tweet.
  • #2775425: The Lightning Extender’s redirection feature sent users to a “Drupal is already installed” error page.
  • PR #140: Fixed a PHP notice arising from the media asset preview handler.

Lightning 8.x-1.01¶

19 July 2016

The Lightning 8.x-1.01 distribution contains the following updates:

Module updates¶

  • Acquia Connector to 8.x-1.2
  • Entity Browser to 8.x-1.0-alpha5
  • Media entity to 8.x-1.2
  • Media entity Twitter to 8.x-1.1
  • Video Embed Field to 8.x-1.1
  • Workbench Moderation to 8.x-1.1

Feature¶

  • |acquia-product:ld| is now easier to extend

    Lightning now contains a standard extension method, enabling website builders to include additional configuration information, using the lightning.extend.yml file during install. This file allows website builders to:

    • Define a list of additional modules to enable after Lightning has finished installing.
    • Add a path that users should be redirected to after the installation of Lightning and any additional modules is complete.

Changes¶

  • Improved CI now automatically tests update path.
  • Lightning Media no longer clears the render cache after entity operations. (#2759313)
  • Updated the front page view to use the [site:name] token instead of printing “Lightning”. (#2757351)
  • Removed custom step definitions which are now included in mink. (#2718123)

Fixes¶

  • Mega patch for Panels included binary files and would fail to apply in certain environments. (#2752375)
  • Video Entity thumbnails are now regenerated if the referenced video is updated. (#2752429)
  • Fixed database update that would occur if updating directly from RC6 to 1.00. (GH #133)
  • Fixed an issue where all media was displayed in the library, regardless of the value of “Save to Library”. (#2757473)
  • Fixed an issue that prevented users from selecting certain cached elements from the media library. (#2757481)

Lightning 8.x-1.00-rc6¶

29 June 2016

The Lightning distribution 8.x-1.00-rc6 release contains the following updates:

Module updates¶

  • Updated Chaos tool suite (ctools) to 8.x-3.0-alpha26
  • Updated Media entity to 8.x-1.1
  • Updated Media entity image to 8.x-1.1
  • Metatag - 8.x-1.0-beta9
  • Updated Video Embed Field to 8.x-1.0
  • Added Views Infinite Scroll 8.x-1.0

The following modules were removed:

  • Services

Changes¶

  • Lightning Media no longer depends on or references the Lightning profile directly.
  • Website builders can choose to opt out of Lightning-generated user roles for content types. These can be changed at Config > System > Lightning.

30 June 2016

Drupal core and module updates¶

  • Udpated Drupal core to 8.1.3
  • Added Entity Browser 8.x-1.0-alpha3
