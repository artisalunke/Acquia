Configuring your Acquia DAM system preferences¶

To configure the preferences for your Acquia DAM implementation, complete the following steps:

  1. Sign in to Acquia DAM with administrative privileges.
  2. In the top right of the page, click the settings icon .
  3. Clicking System Preferences.

Acquia DAM will display the system preferences page that you can use to configure your image and user handling options.

Available settings¶

The Acquia DAM system preferences page provides access to the following functions, by section:

  • General Settings
  • Users and Groups
  • Assets
  • Supported File Extensions
  • Folders
  • Downloads
  • Search
  • Image Download Presets
  • Video Download Presets
  • Cloud Sharing Settings
  • Asset Approval

General Settings¶

  • Admin email notification address – Email notifications listed in the system preferences (such as when new users register, email notification of upload and download, etc.) will be sent to this email address. If you will have multiple account administrators who will need to receive these notifications, we recommend creating an email distribution list—such as dam@example.com—that includes all admins.
  • Enable Early Access for product features on this account check box – Early Access is a great way to stay on the cutting edge of new Acquia DAM features and capabilities. When enabled, Early Access features will automatically be enabled for the account.
  • I would like to learn about participating in Beta programs for product features check box – Participating in Acquia DAM Beta programs allows you to have a voice in the evolution of Acquia DAM products. By selecting this check box, you agree to be contacted by the Acquia DAM Product team to participate in upcoming Beta tests. As a participant, you’ll gain early access to new features while also agreeing to provide feedback on performance.

Users and Groups¶

  • Allow users to self-register check box – When enabled, a registration link will display on the sign in page. This will allow new users to register for an account.
  • Notify when new user registers check box – When enabled, an email notification with the registrant’s information will be sent to the admin email notification address.
  • New users must be approved after registering check box – When enabled, users registering for an account will be added as inactive. This means they will not be able to sign in to Acquia DAM until after an admin activates them. For more information, see Managing users and groups. You can also use this setting for customers with authentication integration. When enabled, users accessing Acquia DAM for the first time through the company’s authentication system will be added to Acquia DAM as inactive.
  • Automatically expire passwords after 90 days check box – When enabled, users will need to reset their password every 90 days.
  • Display the following click-on agreement on registration – Click the edit link to enter terms and conditions that users must agree to when registering for an account. (Read more about terms and conditions sample language.)
  • Session expiration time list – Click the value for the duration of inactivity that can occur before the system requires the user to sign in again.

Assets¶

  • Activate assets upon creation – Set the status of newly uploaded assets to active or inactive. In most instances, the status will be active. Contact your customer success manager for best practices.
  • Default asset name on upload – Set the displayed asset name to match the file name or pull from the headline metadata field.
  • Auto rotate assets on upload check box – When enabled, assets will automatically be rotated to the correct orientation during the upload process.
  • Receive email notification of uploads check box – When enabled, an email will be sent to the admin email notification address when new assets are uploaded to Acquia DAM.
  • Version uploaded assets with same name check box – For information about this option, see Asset version control.
  • Apply metadata from – Select from the New version or Current version options. For information about this option, see Asset version control.
  • Allow suggested keywords check box – For information about this option, see Configuring the metadata schema.
  • Enable keyword taxonomy check box – For information about this option, see Configuring the metadata schema.
  • Enforce keyword taxonomy check box – For information about this option, see Configuring the metadata schema.
  • Deactivate asset upon expiration check box – When enabled, assets will automatically become inactive on their expiration date. If not enabled, assets will remain active after expiration but will not be downloadable by any non-admin user.
  • Receive email upon asset expiration check box – If this option is enabled, an email will be sent to the admin email notification address when the asset expires.
  • Receive email reminders before asset expires - Enable this option if you want to be notified of an asset’s upcoming expiration. Then, enter the number of days prior to expiration that you’d like to be notified.
  • Source of video thumbnail – In the list, click the value to use for which second of the video you want displayed as the thumbnail.

Folders¶

  • Default status of folders on create - Set whether you want the status of newly created folders to be active or inactive. (Read more about the folder statuses.)
  • Keep folders upon FTP import - When enabled, the folders will remain in the FTP client after import. (Read more about uploading assets using FTP.)

Downloads¶

  • Comp (watermarked) downloads:
    • Set to Allow download for all users to have the ability to download watermarked versions of assets.
    • Set to Allow download for logged in users only for signed-in users (therefore not guest users) to have the ability to download a watermarked version of the asset.
    • Set to DO NOT allow download if you don’t want any users to download a watermarked version of the asset.

    Note

    Add a watermark to an asset by ensuring that Watermark assets in this folder is enabled for the asset’s folder. After doing so, click the Edit icon on the asset thumbnail and choose Recreate Thumbnail.

  • Click-on agreement for downloads:

    • Enable click-on agreement - When enabled, users must agree to terms and conditions before downloading assets.
    • Require user to agree:
      • When set to Upon first download only, users must agree to the download terms and conditions the first time they download an asset.
      • When set to For every download, users must agree to the download terms and conditions every time they download an asset.
    • Click-on agreement text - Click edit to add the terms and conditions language. (Read more about Terms and Conditions sample language.)

  • Receive email notification of downloads - When enabled, an email will be sent to the admin email notification address when assets are downloaded.

    • This setting is global and will apply to new folders created in Acquia DAM. If you change the settings, you will also need to change it at the folder level for it to apply to previously created folders.

  • Enable download requests - When this feature is enabled and a group’s download permissions are set to Do not allow, users can request access to download assets. (Read more about editing the permissions of a group.)

    • This setting is global and will apply to new folders created in Acquia DAM. If you change the settings, you will also need to change it at the folder level for it to apply to previously created folders.

Image Download Presets¶

Users can convert images to a different file format, resolution, size and/or color space upon download. Download presets allow you to create predefined options for your users to choose from when downloading the assets.

Create:

  1. Click Create New Image Preset.
  2. Enter a preset name.
  3. Set the file format, resolution, dimensions and/or color space.
  4. Click Save.

Additional Information¶

  • Edit a preset - Click the edit icon .
  • Delete - Click the trashcan icon .
  • If an option is left blank, the file will retain its original setting when it is downloaded.
  • Best practices:
    • Resolution - The recommended values are 72 dpi for web and 300 dpi for print.
    • Color Space - RGB is recommended for web use and CMYK is recommended for print.

Video Download Presets¶

Users can transcode videos upon download. Download presets allow you to create predefined options for your users to choose from when downloading the assets.

  1. Click Create new video preset.
  2. Enter a preset name.
  3. Click the dropdown next to Load preset to work off a preformatted preset. Once selected, the configurations will automatically update.
  4. Select the Format from the dropdown.
  5. Configure the video and audio settings as needed.
  6. Click Save.

Additional Information¶

  • Edit a preset - Click the edit icon .
  • Delete - Click the trashcan icon .
  • If an option is left blank, the file will retain its original setting when it is downloaded.
