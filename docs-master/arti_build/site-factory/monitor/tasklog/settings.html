Configuring task log settings¶

Acquia Cloud Site Factory offers several levels of task log retention, which are configured in the Task log settings page of the Factory interface.

Logging levels and definitions¶

Acquia Cloud Site Factory will retain the log messages for a task that match or exceed the configured logging priority until the task completes. Whether the task ends in failure or success determines which logs are kept after the task ends:

  • Failed tasks - No logs are truncated, and logs are captured and kept based on the configured settings.
  • Successful tasks - Some logs are captured and kept, based on the configured settings.

The following logging levels are listed from most serious (FATAL) to least serious (TRACE):

Name Description Priority
FATAL An unrecoverable error 1 (most)
ERROR A recoverable error — it may be possible to retry the operation and proceed 2
ALERT System events that are unusual, but are not errors (similar to NOTICEin PHP) Examples: A user pauses a task, or a task receives info from an asynchronous process 3
FLOW Changes in state made as part of normal operations 4
WARN A non-essential part of a task did not complete as expected, although the overall task succeeded Example: Errors in user-created hooks 5
INFO Actions taken as part of normal operations 6
DEBUG Diagnostic information potentially useful to both website maintainers and Acquia Support 7
TRACE More verbose diagnostic information, generally useful only to Acquia developers 8 (least)

When determining which logs to keep, Acquia Cloud Site Factory will keep the logs for the selected level and for all levels greater than the selection. For example, selecting the ALERT level for logging will keep all log messages with a status of ALERT, ERROR, and FATAL.

For more information about the types of logging available and how to access them, see Logging in |acquia-product:edg|.

Configuring log message retention¶

To configure how Acquia Cloud Site Factory retains generated log messages, complete the following steps:

  1. Sign in to the Factory interface with either the `release engineer </site-factory/users/admin/release-engineer>`__ or `developer </site-factory/users/admin/developer>`__ role.
  2. In the admin menu, click Administration, and then click the Task log settings link.
  3. Use the following lists to configure the log filtering for your Acquia Cloud Site Factory-hosted websites, based on the available logging levels:
    • Logging level for successful tasks - The default threshold of log retention after a task completes successfully — Recommended value: ``WARN``
    • Logging level for completed tasks - The default threshold of log retention, with log messages with a status less than the configured threshold not being saved — Recommended value: ``INFO`` (unless you are investigating problems with your website)
  4. Click Save configuration.
