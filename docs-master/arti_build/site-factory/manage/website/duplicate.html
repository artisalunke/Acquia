Duplicating a site¶

Acquia Cloud Site Factory enables you to create duplicate websites from an existing website. These duplicate websites can either contain only the structure of the original website (clean copy), or can be complete and exact copies of the original website (full website clone).

Duplicating your website can be useful in the following situations:

  • You need to create several websites with the same or similar configurations.
  • You have a production website that you’d like to experiment with, but you want to make changes that external website visitors should not encounter.

Note

If you duplicate a website with a custom domain, Acquia Cloud Site Factory removes any custom domain redirection settings from the duplicated website.

What gets duplicated¶

When you duplicate a website, depending on the option that you select, the new website will contain the following:

Comparison of duplication methods¶

Description Clean copy Full website clone
Configurations All website configurations All website configurations
Content: General Content created by anonymous users, and user accounts with the administrator or website maintainer roles All content
Content: Comments Not copied All content
Database tables All tables are copied, but tables with ephemeral data, such as the sessions, accesslog, semaphore, batch, and queue tables are truncated All tables
Files Files uploaded to the website’s Media Library or attached to content using the File field. Files owned by removed users are reassigned to the Anonymous user. Files uploaded to the website’s Media Library or attached to content using the File field
Modules All modules copied. Some disabled modules may be re-enabled if they are needed during the user scrubbing process. All modules
Structure All structural elements, including blocks and taxonomies All structural elements, including blocks and taxonomies
Themes All themes All themes
Users User 1, all user accounts with the administrator or website maintainer roles, all OpenID administrator users, and any users listed in a hook that preserves accounts from scrubbing All user accounts
All other items Not copied Copied to the new website

Duplicating a website¶

To duplicate an Acquia Cloud Site Factory website, use one of the following methods:

  • Using the /sites/[siteID]/duplicate endpoint in the Acquia Cloud Site Factory API
  • Using the Site Factory Management Console

To duplicate a website through the Site Factory Management Console user interface:

  1. Sign in to the Site Factory Management Console, find the website that you want to duplicate, and then open its actions menu.

  2. Click Duplicate site.

  3. In the text field, enter the new Acquia Cloud Site Factory name for your website.

  4. Select the duplication option that you want to use for your website:

    • Create a clean copy
    • Full site clone

  5. Click Duplicate site.

The duplication process begins and can take several minutes to complete. When it’s finished duplicating, Acquia Cloud Site Factory sends a completion email to users with the platform admin and site builder roles.

Duplicate websites and groups

Websites duplicated from the All my sites page will not be part of any group or subgroup.

If you duplicate a website from within a group, the new website will be part of that group. It will not inherit assigned groups and subgroups — the duplicate website is assigned only the group or subgroup in which you started the duplication process.

Because the website duplication process copies the existing user account information, members of the original website can sign in on the duplicate website using their same username and password.
