Site Factory Management Console roles¶

In order to manage your Acquia Cloud Site Factory environment and websites, you must sign in with an account that has the appropriate permissions for the actions that you need to accomplish. Acquia Cloud Site Factory permissions are gathered in roles, which enable administrators to assign a collection of permissions to an account, instead of assigning them one at a time. Your account can be assigned one or more of the following Acquia Cloud Site Factory roles in the Site Factory Management Console:

  • platform admin - Provides overall responsibility for Acquia Cloud Site Factory deployment, including management of websites and OpenID accounts
  • site builder - Allows for website and content creation and management
  • developer - Enables updating development and staging environments with new and updated code
  • release engineer - Grants access to the PaaS production environment for code pushes and update status monitoring

In addition to the individual roles, you can also assign websites to groups, and manage access to the websites in those groups by adding users to a group. For more information, see Managing site group users.

Comparing website roles and Site Factory Management Console roles¶

Acquia Cloud Site Factory and the websites in an Acquia Cloud Site Factory deployment can have different sets of user roles. For example, a user might have the platform admin role on Acquia Cloud Site Factory, but only be a standard registered member on one of the Acquia Cloud Site Factory websites.

A user’s role on the website governs the level of access the user has on the website. Therefore, a platform admin on Acquia Cloud Site Factory would need administrator privileges on the website itself to maintain a high level of access.

You can use centralized role management to map Acquia Cloud Site Factory platform admin and site builder roles to hosted website roles. For example, if you have a role on your hosted websites named content manager, you can specify that all users with the platform admin role also have the content manager role on each hosted website. For more information, see Assigning hosted site roles during sign-in.
