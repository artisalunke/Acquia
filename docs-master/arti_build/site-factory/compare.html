Acquia Cloud Site Factory and Drupal¶

If you already know Drupal, you may wonder what benefits and features Acquia Cloud Site Factory adds.

Although Acquia Cloud Site Factory is built using Drupal, it provides additional tools and services to help you focus on building sites, instead of how to build sites.

The following list explains some of the differences between Acquia Cloud Site Factory and a stock Drupal website:

  • Hosting and monitoring - Acquia Cloud Site Factory provides a high-availability hosting and monitoring infrastructure across multiple web servers, with Varnish caching and replicated database servers. An operations team is on call 24/7 to test your site and keep it up-to-date with all the relevant updates and security patches. All of this is based on the same Acquia Cloud Enterprise infrastructure that powers and protects some of the largest and most active Drupal sites in the world. With Acquia Cloud Site Factory, you can stop worrying about site speed, site traffic, disk space, up-time, and backups.
  • Single sign-on, multisite management - Instead of signing in to multiple websites, if they’re on Acquia Cloud Site Factory, sign in once to Acquia Cloud Site Factory to sign in to all of them. This helps users reduce the number of accounts and passwords they have to keep track of, and it allows Acquia Cloud Site Factory website administrators to view and manage all of their websites from a single interface.
  • Acquia Support - Acquia Cloud Site Factory provides you with access to Acquia support, providing you with professional Drupal support and how-to guidance, even for custom CSS. Accounts are also entitled to support tickets, where they can have one-on-one private support conversations.
  • Open software-as-a-service (SaaS) - There’s no vendor lock-in with Acquia Cloud Site Factory. If you decide that you want to move your website to another hosting service, or even host it yourself, you can export your website at any time.
