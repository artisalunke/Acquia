Using GitHub with the pipelines client¶

Acquia Cloud pipelines client

  • Intro
  • Install
  • Connect
  • Require
  • Commands
  • Migrate
  • Workflows

By default, the pipelines client uses your Acquia Cloud code repository as the source repository for build jobs. As an alternative, however, you can configure the client to use a GitHub repository as the source, so the command-line client runs a build job and commits the build artifact to your Acquia Cloud target repository when commits are pushed, or pull requests are created.

You can start with an Acquia Cloud code repository and clone it into GitHub, or you can start with a GitHub repository and push it into the Acquia Cloud code repository.

Note

Installing the pipelines command-line client is optional, and is not required for pipelines to function with your application. To connect through the user interface, see Connecting an application to an external Git repo.

Getting started¶

Use the following pages to help you use your GitHub repositories with the pipelines client:

  • Configuring the GitHub connection to the pipelines client
  • Obtaining required GitHub information
  • Migrating a repo from |acquia-product:ac| to GitHub

If you prefer to connect to GitHub from the user interface, see Connecting an application to an external Git repo.
