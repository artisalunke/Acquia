2017 Release notes - Acquia Cloud¶

Looking for the latest and greatest new features and changes to the Acquia Cloud platform? Read on and check back regularly to see what else we’ve done.

Acquia Cloud updates - week of December 25, 2017

Tue, 2 Jan 2018

The Acquia Cloud product includes the following updates for the week of December 25, 2017 through December 31, 2017:

Change

  • When signing in, non-hosted customers are now directed to Acquia Insight tools in the Acquia Cloud interface, rather than the Environments page.

Acquia Cloud interface - week of December 18, 2017

Tue, 26 Dec 2017

The Acquia Cloud interface includes the following updates for the week of December 18, 2017 through December 25, 2017:

Changes

  • The Varnish cache settings in the Acquia Cloud interface now check all domains by default, and remember previous settings.
  • The Stack Metrics page, now supports specific or custom time ranges.
  • Links in the Acquia Cloud interface now all correctly reference cloud.acquia.com.
  • Environment cards have been updated for consistency.

Fixed issues

  • The source and target branch selections for Bitbucket builds in the Acquia Pipelines feature are now consistent with other builds.

Acquia Cloud interface - week of December 11, 2017

Mon, 18 Dec 2017

The Acquia Cloud interface includes the following updates for the week of December 11, 2017 through December 17, 2017:

Changes

  • The circle components of Stack Metrics graphs have updated colors for better accessibility.
  • Applications now list the number of CD environments used by that application.
  • Elastic IP addresses will no longer appear on the Servers page unless they are specifically assigned to that environment.

Acquia Cloud interface - week of December 4, 2017

Tue, 12 Dec 2017

The Acquia Cloud interface includes the following updates for the week of December 4, 2017 through December 12, 2017:

Feature

  • Updated |acquia-product:ais| interface with multisite support Acquia Insight has been updated with a new interface that supports both individual websites and multisite installations, with easy-to-read graphs and statistics to help you monitor performance across all of your websites and, giving you added visibility into the various applications that may be sharing code and databases. With this updated feature, all Acquia Insight scores have been removed from the Workflow and Environment pages and are consolidated in the Acquia Insight page. Learn more.

Changes

  • The Acquia Cloud API implemented CORS (Cross-origin resource sharing), allowing for API authentication within the browser.
  • Acquia Cloud CD environments can now be deleted from environment listing page.

Fixed issue

  • Acquia Cloud Pipelines jobs were not correctly limited to two concurrent jobs.

Acquia Cloud interface - week of November 27, 2017

Mon, 4 Dec 2017

The Acquia Cloud interface includes the following update for the week of November 27, 2017 through December 3, 2017:

Changes

  • Subversion (SVN) is no longer supported or available in any Acquia Cloud user interface pages. Git should be used for version control; it is not possible to switch to SVN.
  • Where applicable, the elastic load balancer (ELB) name is visible on the Domains page.
  • On the Environments page, environments now are listed in this order:
    • Dev
    • Stage
    • Prod
    • Custom environments
    • Remote Administration (RA) environments
    • Acquia Cloud CD environments
  • For Acquia Cloud Professional customers, the Stack Metrics page now displays a set of graphs summarizing overall server health.

Acquia Cloud 1.103

Mon, 16 Oct 2017

Acquia Cloud includes the following updates:

Release schedule

Platform Release Date
Acquia Cloud Professional Monday, October 23
Acquia Cloud Enterprise Wednesday, November 1
Acquia Cloud Site Factory Wednesday, November 8

Changes

Important

Beginning November 15, 2017, all new websites deployed on Acquia Cloud will default to PHP 7.1.

  • The following included software was updated:
    • Updated PHP to version 7.1.10
  • Drush 6 is the default Drush version.
  • Drush 8 is available, but is not the default version. While it was temporarily the default after 1.103 was released, Acquia has reverted this change until we resolve an issue with the daily database backups for customers who also have a composer-included vendor version of Drush 8.
  • APCu shm_size defaults to 32MB.
  • TLS 1.0 will not be enabled on any new elastic load balancers (ELBs).
  • Several internal updates were made to Acquia Cloud relating to general stability improvements.

Acquia Cloud 1.102

Wed, 23 Aug 2017

Acquia Cloud includes the following updates:

Release schedule

Platform Release Date
Acquia Cloud Professional Thursday, August 24
Acquia Cloud Enterprise Wednesday, August 30
Acquia Cloud Site Factory Wednesday, September 6

Changes

  • The following included software was updated:
    • Updated New Relic to 7.4.0.198
    • Updated Ngnix to 1.12.1
    • Updated PHP versions to 5.6.31, 7.0.21, and 7.1.7
    • Updated Varnish to version 4.1.8
  • New elastic load balancers are provisioned with IPv6 support. (CL-16245)
  • Apache error logs now include the following additional fields for debugging:
    • pid
    • X-AH-Client-IP
    • Referer
    • User-agent
    • vhost
    • X-Forwarded-For
    • request_id
  • The APCu size has been increased to 32Mb for newly deployed websites. (CL-14571)
  • PHP 7.0 now requires the use of Drupal 7.55 or greater.
  • The Acquia Cloud migration workflows were improved to decrease downtime. (CL-18134)
  • Drush aliases now meet Drupal coding standards, reducing unnecessary errors. (CL-15888)
  • The transaction logs for New Relic no longer include ACQUIA_MONITOR, AH_VIEW, or AH_DOWNLOAD.
  • Acquia Cloud no longer supports the following features or items:
    • TLS 1.0 (CL-17403)
    • Traceview (CL-14773)
    • Usage of port 8080 (CL-17026)

Fixed issues

  • When upgrading to PHP 7.1, Xdebug caused errors. (CL-18693)
  • Mysqldump backups did not store utf8mb4 characters, and instead replaced them with question marks. (CL-14004)
  • In some circumstances, backups could fail for Acquia Cloud users. (CL-18124, CL-15741, CL-18432)

Acquia Cloud 1.100

Thu, 11 May 2017

Acquia Cloud includes the following updates:

Release schedule

Platform Release Date
Acquia Cloud Professional Thursday, May 11
Acquia Cloud Enterprise Wednesday, May 17
Acquia Cloud Site Factory To be determined

Upcoming change to supported Drush versions

Acquia Cloud version 1.101 will remove support for both Drush 3 and Drush 4.

Changes

  • The MySQL sql_mode setting override for multiregion Drupal 8 websites is available for use on Acquia Cloud. (CL-15272)

  • If a file migration attempt exceeds the quota for Acquia Cloud Free- or Acquia Cloud CD-based environments, Acquia Cloud will stop the process and display an error message. (CL-16346)
  • Several modifications were made to Acquia Cloud to both improve general reliability and shorten maintenance windows.

Acquia Cloud 1.97

Thu, 26 Jan 2017

Acquia Cloud includes the following updates:

Release schedule

Platform Release Date
Acquia Cloud Professional Thursday, January 26
Acquia Cloud Enterprise Wednesday, February 1
Acquia Cloud Site Factory Wednesday, February 8

Changes

  • Upgraded Drush to version 8.1.8. (CL-14669)

    Important

    You cannot use Symfony 3.x or greater with Drush 8.1.8 in Acquia Cloud applications. To work around this issue, use one of the following options:
