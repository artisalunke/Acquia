Acquia Dev Desktop folders and files¶

When you install Acquia Dev Desktop and add one or more Drupal websites, the following directories and files are created in your local file system, depending on whether you are using macOS or Windows:

macOS directories and files¶

Directory/File Description
/Applications/DevDesktop/apache /    conf/vhosts.conf The virtual hosts file for all websites known to the Acquia Dev Desktop application.
/Applications/DevDesktop/    Acquia Dev Desktop.app A package that contains the configuration files for Acquia Dev Desktop.
~/Sites/devdesktop A directory containing Acquia Dev Desktop websites.
~/.acquia/backup When you upgrade to a newer version of Acquia Dev Desktop, your existing installation is backed up to this directory.
~/.acquia/DevDesktop/    DrupalSettings The website configuration files included by settings.php.
~/.acquia/DevDesktop/Drush/Alia ses/    aliases.drushrc.php Drush aliases for local and Acquia Cloud websites (downloaded from Acquia Cloud). These aliases include the host, docroot, user, site, and environment of the Acquia Cloud website.
~/.acquia/DevDesktop/Drush/Alia ses/    cl.[sitename].aliases.drushr c.php Drush aliases for all remote Acquia Cloud-hosted websites known to Acquia Dev Desktop. These aliases include the host, docroot, user, site, and environment of the website.

Any passwords or user names in these files are encrypted to protect your security.

If you chose something other than the default locations for the Acquia Dev Desktop stack location and sites directory when you installed Acquia Dev Desktop, these paths will vary, based on the locations you chose.

Note

In macOS, the /.acquia directory is hidden. To locate the directory, complete the following steps:

  1. Open Finder.
  2. In the menu bar, select Go > Go to folder.
  3. Enter the path of the directory, and then click Go.

Windows directories and files¶

Directory/File Description
C:\Program Files (x86)\Dev Desk top\    apache\conf\vhosts.conf The virtual hosts file for all websites known to the Acquia Dev Desktop application.
C:\Program Files (x86)\Dev Desk top\    AcquiaDevDesktop\AcquiaDevDe sktop2.exe The executable file for Acquia Dev Desktop.
C:\Users\[username]\Sites\    devdesktop A directory containing Acquia Dev Desktop websites.
C:\Users\[username]\.acquia\    backup When you upgrade to a newer version of Acquia Dev Desktop, your existing installation is backed up to this directory.
C:\Users\[username]\.acquia\    DevDesktop\DrupalSettings The website configuration files included by settings.php.
C:\Users\[username]\.acquia\Dev Desktop\    Drush\Aliases\[sitename].ali ases.drushrc.php Drush aliases for local and Acquia Cloud websites (downloaded from Acquia Cloud). These aliases include the host, docroot, user, site, and environment of the Acquia Cloud website.
C:\Users\[username]\.acquia\Dev Desktop\    Drush\Aliases\    cl.[sitename].aliases.drushr c.php Drush aliases for all remote Acquia Cloud-hosted websites known to Acquia Dev Desktop. These aliases include the host, docroot, user, site, and environment of the website.
C:\Documents and settings\[user name]\.acquia\DevDesktop\    DrupalSettings\loc_[sitename ]_dd.inc Basic configuration settings for each environment in Acquia Dev Desktop.

Any passwords or user names in these files are encrypted to protect your security.

If you chose something other than the default locations for the Acquia Dev Desktop stack location and sites directory when you installed Acquia Dev Desktop, these paths will vary, based on the locations you chose.
