2016 Release notes - Profile Manager¶

Looking for the latest and greatest new features and changes to Profile Manager? Read on and check back regularly to see what else we’ve done.

Profile Manager 1.40¶

13 December 2016

This release of the Profile Manager product contains several modifications that will not impact users’ experiences with the product.

Profile Manager 1.39¶

7 December 2016

The Profile Manager product contains the following updates:

Change¶

  • The Manage containers page has been renamed to Manage JavaScript. (CH-5504)

Fixed issues¶

  • The ability to manage JavaScript was incorrectly removed from the Profile Manager interface. (CH-5504)
  • The Person Details page in Profile Manager displayed duplicate Last Error and Last Error Date fields. (CH-5488)

Profile Manager 1.38¶

22 November 2016

The Profile Manager product contains the following update:

Change¶

  • For clarity, the Configuration Data link on the Admin > Manage configuration data webpage was changed to Customer sites. (CH-5378)

Profile Manager¶

8 November 2016

The Profile Manager product contains the following updates:

Note

The [acquia-product:alw] product has been renamed to Profile Manager, and is a part of [acquia-product:cha], along with [acquia-product:leb] and [acquia-product:ch].

Changes¶

  • Users already signed in to Profile Manager are now also signed in to [acquia-product:leb] (and vice versa) using a temporary token cookie. (CH-5391)
  • The Profile Manager administrative interface (previously [acquia-product:alw]) has been updated with its new name. (CH-5379)
  • The latitude and longitude Touch fields were changed to use a float data type. (CH-5335)

Fixed issue¶

  • Some custom-defined fields did not properly respect case-insensitivity. (CH-5410)
