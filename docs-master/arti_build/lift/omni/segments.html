Data warehouse Segment field types¶

Acquia Lift data warehouse

  • Intro
  • Person
  • Person
  • Touch
  • Event
  • Segments

The data warehouse uses the Segments field layout to describe segments associated with each event for a given person.

The Segments field layout includes the following tables:

  • segment
  • matched_segment

segment table¶

The Segment table contains a list of the segments in Acquia Lift and is synchronized with the configuration database.

Field name Data type Sortable Description
id INT (4 bytes) Unique segment ID — Acquia Lift internal value
name VARCHAR (100) Name of the segment (for example, All article pages)
tracking_id VARCHAR (100) Shortened machine name of the segment name, from the Profile Manager segment builder

matched_segment table¶

The Matched_Segment table contains a list of the events and segments that are associated with each segment and links the Segments table to the Event table.

Field name Data type Sortable Description
event_id INT (8 bytes) Link to an existing ID for an event in the Event table — Acquia Lift internal value
person_id INT (8 bytes) Link to an existing id for a person — Acquia Lift internal value
segment_id INT (4 bytes) Link to an existing ID for a segment in the Segments table — Acquia Lift internal value
db_last_modified_date TIMESTAMP The UTC time and date when the row was last modified in the database
